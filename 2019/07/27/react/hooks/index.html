hooks

Created2019-07-27|Updated2020-12-12
|Post View:

Hook 是react16.8的新增特性，它可以让你在不编写class的情况下使用state以及生命周期特性的函数

解决问题

  1. 无class的复杂性，函数式组件，比class更容易理解。
  2. 无生命周期的困扰，class组件生命周期太多太复杂，使用函数组件存在状态。
  3. 避免地狱嵌套，可读性提高
  4. 解决HOC和Render props的缺点
  5. 优雅的复用，UI和逻辑更容易分离

useEffect

useEffect可以让我们在函数组件中执行副作用操作。事件绑定，数据请求，动态修改DOM。

useEffect第二个参数为一个数组。当我们提供第二个参数时，只有第二个参数被更改useEffect才会执行。利用第二个参数我们可以模拟出类组件的生命周期函数

useEffect注意点：

1.React首次渲染和之后的每次渲染都会调用一遍useEffact函数。

2.useEffect中定义的函数的执行不会阻碍浏览器更新视图，也就是说这些函数是异步执行的。

  • 而componentDidMoount和componentDidUpdate中的代码都是同步执行的

3.useEffect的第二个参数：dependences

  • dependences这个参数定义了useEffect的依赖，在新的渲染中，只要所有依赖项的引用都不会发生变化，useEffect就不会被执行，而且当依赖项为[]，useEffect仅在初始化执行一次，后续的Rerender永远也不会被执行。

4.useEffect实现componentWillUnmount生命周期函数

  • 当第二个参数传空数组[]时，就是当组件将被销毁时才进行解绑，这也就实现了componentWillUnmount的生命周期函数

5.开发者通过useEffect的第二参数告诉react用到了那些外部变量。

6.多个useEffect的时候，注意顺序。

useLayoutEffect

useLauoutEffect是组件DOM渲染出来之前执行的，相当于useEffect的拦截

  1. 在useEffect之前执行
  2. useEffect:异步，不会阻塞浏览器的重绘
  3. useLauyoutEffect会阻塞浏览器的重绘。如果你需要手动的修改DOM，推荐使用useLauoutEffect

useContext

useContext可以帮助我们跨越组件层级直接传递变量，实现共享，需要注意的是useContext和redux的作用是不同的，一个解决的是组件之间值传递的问题，一个是应用中统一管理状态的问题，但通过和useReducer的配置使用，可以实现类似Redux的作用

useReducer

useReducer接受三个参数，reducer函数，initialArg初始化，init惰性初始值函数。

reducer函数和redux的reducer类似，接受state，以及action。返回更新后的state。如果传入三个参数，init(initialArg)将作为初始值。

useState继承于useReducer.

1.实际在useState内部，也是调用useReducer来创建状态和set函数。

2.useReducer(函数，初始状态)，接受触发函数和当前应用state，执行结果并返回最新state.

useRef

useRef的作用

1.获取DOM元素节点

2.获取子组件的实例

3.渲染周期之间共享数据的存储(state不能存储跨渲染周期的数据，因为state的保存会触发组件重渲染)
