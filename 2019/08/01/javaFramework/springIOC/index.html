spring-ioc

spring核心功能包括IOC，AOP编程，持久层JDBCtemplate，表示层MVC，事务处理

什么是IOC

  • 把对象创建和对象之间的调用过程，交给Spring进行管理
  • 使用IOC目的：为了降低耦合度

ioc底层使用到：xml解析、工厂模式、反射

ioc原理

使用原始方式创建对象，对象和对象的耦合度比较高

使用工厂模式降低对象之间的耦合度

ioc原理：解析配置文件，通过全类名反射创建class对象。用到xml，反射，工厂模式。

IOC容器的实现方式

Spring提供的IOC容器实现的两种方式（两个接口），作用：加载配置文件，创建bean容器。

BeanFactory接口：IOC容器基本实现是Spring内部接口的使用接口，不提供给开发人员进行使用（加载配置文件时候不会创建对象，在获取对象时才会创建对象。）

ApplicationContext接口：BeanFactory接口的子接口，提供更多更强大的功能，提供给开发人员使用（加载配置文件时候就会把在配置文件对象进行创建）推荐使用！

bean 生命周期

生命周期 ：从对象创建到对象销毁的过程

bean生命周期的过程：

  1. 通过构造器创建 bean 实例（无参数构造）

  2. 为 bean 的属性设置值和对其他 bean 引用（调用 set 方法）

  3. 调用 bean 的初始化的方法（需要进行配置初始化的方法）

  4. bean 可以使用了（对象获取到了）

  5. 当容器关闭时候，调用 bean 的销毁的方法（需要进行配置销毁的方法）

基于注解方式

什么是注解

  • 注解是代码特殊标记，格式：@注解名称(属性名称=属性值, 属性名称=属性值…)
  • 使用注解，注解作用在类上面，方法上面，属性上面
  • 使用注解目的：简化 xml 配置

创建对象的注解

Spring 针对 Bean 管理中创建对象提供一下注解，四个注解功能是一样的，都可以用来创建 bean 实例。

  • @Component

  • @Service

  • @Controller

  • @Repository
